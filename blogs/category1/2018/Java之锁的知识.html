    # 公平锁与非公平锁

    什么是公平锁，什么是非公平锁？其实我们可以知道，在现实生活中，公平就是先到先得嘛，就像排队一样，你先来了就肯定先给你服务，这就是所谓的公平；何为不公平呢？就是即使你来第一，但是依旧插你队，你先来但是你不是先被服务的，那么就是所谓的不公平。

    公平锁：就是没有优先级之说的，先到先服务的锁；

    非公平锁：有优先级，提高吞吐量；

    依旧这个原理：公平锁中我们有 new ReentrantLock()，为什么说该锁是公平锁？

    我们看它的底层源码：

    public ReentrantLock() {
        sync = new NonfairSync();
    }

public ReentrantLock(boolean fair) {
        sync = fair ? new FairSync() : new NonfairSync();
    }

    以上我们得知：Lock是可以根据参数来调整自己是否是公平的锁！如果是false，则为非公平锁，如果是true，则为公平锁，而且在默认的情况下，ReentrantLock默认为非公平锁。

    当然我们熟知的synchronized锁就是一个典型的非公平锁，哪个线程抢到了锁，就可以访问该方法。（该篇文章比较详细讲了 synchronized锁的是哪些对象 (opens new window)）

    # 可重入锁（递归锁）

    # 理论

    啥是可重入锁？

    指的是同一线程外层函数拿到锁之后，在进入内层方法会自动获取锁。（类似于 你的房子 只需要一把锁，你打开了大门的锁，就可以进入你的厨房，你家的厕所，不需要再去拿锁开锁开门吧？）

    # 代码

    我们有一个Phone类：

    class Phone {
    public synchronized void sendSMS() {
        System.out.println(Thread.currentThread().getName() + "发送短信");
        // 这里嵌套了一个方法
        sendEmail();
    }

    private synchronized void sendEmail() {
        System.out.println(Thread.currentThread().getName() + "发送email");
    }
}

    # 自旋锁

    # 理论

    之前我们学volatile的时候，其实就已经接触过自旋的思想了：还记得那个getAndAddInt()方法嘛？底层就是：

    public final int getAndAddInt(Object o, long offset, int delta) {
     int v;
     do {
         v = getIntVolatile(o, offset);
     } while (!compareAndSwapInt(o, offset, v, v + delta));
     return v;
 }

    这其实就是一个自旋操作。

    所谓自旋：尝试获取锁的线程不会阻塞，而是采用循环的方式去尝试获取锁，这样的好处就是减少线程上下文切换，但是缺点是比较消耗CPU。自旋的本质就是 while循环 + CAS

    # 代码

    public class SpinLockDemo {
    // 原子引用线程
    AtomicReference<Thread> atomicReference = new AtomicReference<>();

    public static void main(String[] args) throws InterruptedException {
        SpinLockDemo spinLock = new SpinLockDemo();
        new Thread(() -> {
            try {
                spinLock.MyLock();
                // 在这期间，线程t2没办法拿到锁（不断CAS自旋操作尝试获取锁），就需要等待线程t1释放锁
                Thread.sleep(5000);
                spinLock.MyUnLock();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }, "t1").start();
        // 保证线程t1先拿到锁
        Thread.sleep(1000);
        new Thread(() -> {
            try {
                spinLock.MyLock();
                Thread.sleep(1000);
                spinLock.MyUnLock();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }, "t2").start();
    }
    public void MyLock() {
        // 获取当前进来该方法的线程
        Thread thread = Thread.currentThread();
        System.out.println(thread.getName() + "\t come in lock ~");
        while (!atomicReference.compareAndSet(null,thread)){

        }
    }
    public void MyUnLock() {
        Thread thread = Thread.currentThread();
        System.out.println(thread.getName() + "\t come in unlock");
        atomicReference.compareAndSet(thread, null);
    }
}

    # 读写锁

    # 理论

    什么是读写锁呢？

    读写锁分为读锁和写锁：

    **读锁：**多个线程同时读一个资源类没有问题，所以为了满足并发量，读取共享资源应该可以同时进行；

    **写锁：**如果一个线程去写共享资源，就不应该再有其他线程来对该资源的读取或者写入。

    # 代码

    代码验证：

    自定义的缓存MyCache：

    class MyCache {
    private volatile Map<String, Integer> map = new HashMap<>();

    public void put(String key, int value) {
        System.out.println(Thread.currentThread().getName() + "正要写入的key为：" + key);
        map.put(key, value);
        System.out.println(Thread.currentThread().getName() + "写入成功");
    }

    public void get(String key) {
        System.out.println(Thread.currentThread().getName() + "正在读取");
        int value = map.get(key);
        System.out.println(Thread.currentThread().getName() + "读取成功：" + value);
    }
}

    # CountDownLatch

    相当于计数器：用一个口诀记住：秦灭六国，一统华夏！（也就是说 只有完成消灭六国，秦国才是一统华夏）

    public class CountDownLatchDemo1 {
    public static void main(String[] args) throws InterruptedException {
        // 约定总共计数器为6
        CountDownLatch countDownLatch = new CountDownLatch(6);
        for (int i = 1; i <= 6; i++) {
            final int num = i;
            new Thread(() -> {
                System.out.println("第" + num + "号同学 \t 走出教室");
                // 计数器 -1
                countDownLatch.countDown();
            }, "t" + i).start();
        }
        // 这里必须要加个await()方法，让main线程进行等待
        countDownLatch.await();
        System.out.println(Thread.currentThread().getName() + "班长最后离开教室");

}

    # CyclicBarrier

    一句话：集齐七颗龙珠，才能召唤神龙！

    和CountDownLatch作用相反。

    现在提一个需求：集齐七颗龙珠才能召唤神龙。

    public class cyclicBarrierDemo {
    public static void main(String[] args) {
        CyclicBarrier cyclicBarrier = new CyclicBarrier(7,
                () -> System.out.println("*****召唤神龙"));
        for (int i = 1; i <= 7; i++) {
            final int num = i;
            new Thread(()-> {
                System.out.println(Thread.currentThread().getName() + "\t 收集到第：" + num + "颗龙珠");
                try {
                    // 该线程进行等待
                    cyclicBarrier.await();
                } catch (InterruptedException | BrokenBarrierException e) {
                    throw new RuntimeException(e);
                }
            },String.valueOf(i)).start();
        }
    }
}

    # Semaphore

    顾名思义是信号量的意思。

    两个作用：

    • 一个用于多个共享资源的互斥使用；
    • 一个用于并发线程数量的控制。

    现在有一个需求：六个车对三个车位的分配。

    public class SemaphoreDemo {
    public static void main(String[] args) {
        Semaphore semaphore = new Semaphore(3); // 模拟三个车位
        for (int i = 1; i <= 6; i++) {
            new Thread(() -> {
                try {
                    // acquire()
                    semaphore.acquire();
                    System.out.println(Thread.currentThread().getName() + "\t 抢到车位");
                    Thread.sleep(3000);
                    System.out.println(Thread.currentThread().getName() + "\t 停车3s后离开车位");
                } catch (InterruptedException e) {
                    e.printStackTrace();
                } finally {
                    // release()
                    semaphore.release();
                }
            }, String.valueOf(i)).start();
        }
    }
}
