    Netty传输文件、分片发送、断点续传

    考小拉 2/3/2023 IM系统项目网络

    # 你是如何实现该功能的？

    1. 在传输大文件的时候，要保证其性能，需要做到哪几点？
      • 选择正确的传输协议，比如：protostuff 传输协议；
      • 相对于以前传统文件所用的BIO流传输，以块的方式去传输会更加高效，比如：分片发送
    2. 用Java文件操作的RandomAccessFile类来支持文件的定位读取和写入；
    3. 在启动服务端后，可以向客户端发送文件传输指令；允许接收文件、控制读取位点、记录传输标记、文件接收完成；
    4. 还需要注意一个重要的细节：我们设定的 byte[1024] 为默认读取范围，但当读取到最后的时候可能不足1024个字节，就会出现空字节，那么我们不能读取这空字节，否则我们的文件会出现额外的信息而导致文件打不开。

    下面开始我们这个功能的实现步骤：

    接下来就是实现文件读写工具类 === 》分片读取写入处理类

    具体是如何实现 文件分片读取和写入这个功能：

    • 读取文件

      先创建一个File类文件对象作为RandomAccessFile对象中传入的参数，创建RandomAccessFile类的对象（该类对象主要用来 设置文件的读写模式和 确定文件的读取以及写入的位置（seek方法））并设置 ”r“ 读模式，创建字节数组byte[] 用来存储一次性从文件读取的数据。开始读取的时候，我们会有一个变量readSize 来记录每次读取的数据大小，倘若该数据小于等于0则可以表明该文件读取完毕了；创建FileBurstData类对象（文件分片对象）并给其设置部分属性，此时还需要一次判断readSize的大小，倘若该数组小于1024字节，需要利用System.arraycopy() 拷贝数据到新的数组，然后给FileBurstData类对象（文件分片对象）设置完最后的属性。（代码实现如下）

    • 写入文件

      从文件状态开始判断，如果是COMPLETE 则传输完毕；先设置好客户端那边的存储文件的地址，即创建文件对象 File file = new File(baseUrl + "/" + fileBurstData.getFileName()); ，创建RandomAccessFile类的对象（该类对象主要用来 设置文件的读写模式和 确定文件的读取以及写入的位置（seek方法））并设置 ”rw“ 读写模式，然后就可以 将文件数据写出到指定位置的操作了；此时判断文件状态，如果是END，就表示该文件的分片结束了；创建 fileBurstInstruct 类对象（文件分片指令对象），给该对象设置状态 和 客户端存储文件的地址 和 下一次文件的读取位置即可。

    上述描述 下面代码实现

    public class FileUtil{
    
    // 读取文件
    public static FileBurstData readFile(String fileUrl,Integer readPosition){
        File file = new File(fileUrl);
        //r: 只读模式 rw:读写模式
        RandomAccessFile randomAccessFile = new RandomAccessFile(file, "r");
        // 移动文件记录指针的位置
        randomAccessFile.seek(readPisition);
        byte[] bytes = new byte[1024];
        // 读取到字节数组的大小
        int readSize = randomAccessFile.read(bytes);
        if(readSize <= 0) {
            randomAccessFile.close();
            return new FileBurstData(Constants.FileStatus.COMPLETE);
        }
        // 创建FileBurstData对象并给其设置属性
        FileBurstData fileInfo = new FileBurstData();
        fileInfo.setFileUrl(fileUrl);
        fileInfo.setFileName(file.getName());
        fileInfo.setBeginPos(readPosition);
        fileInfo.setEndPos(readPosition + readSize);
       	// 不足1024需要拷贝去掉空字节
        if(readSize < 1024) {
            byte[] copy = new byte[readSize];
            System.arraycopy(bytes,0,copy,0,readSize);
            fileInfo.setBytes(copy);
            fileInfo.setStatus(Constants.FileStatus.END);
        }else{
           	fileInfo.setBytes(bytes);
            fileInfo.setStatus(Constants.FileStatus.CENTER); 
        }
        randomAccessFile.close();
        return fileInfo;
    }
    
    // 写入文件
     public static FileBurstInstruct writeFile(String baseUrl, FileBurstData fileBurstData) throws IOException {
         if (Constants.FileStatus.COMPLETE == fileBurstData.getStatus()) {
            return new FileBurstInstruct(Constants.FileStatus.COMPLETE);
        }
		// 设置好客户端那边的存储文件的地址
        File file = new File(baseUrl + "/" + fileBurstData.getFileName());
        // r: 只读模式 rw:读写模式
        RandomAccessFile randomAccessFile = new RandomAccessFile(file, "rw");
        // 调用了seek（start）方法，是指把文件的记录指针定位到start字节的位置。也就是说程序将从start字节开始写数据
        randomAccessFile.seek(fileBurstData.getBeginPos());
        randomAccessFile.write(fileBurstData.getBytes());        
        randomAccessFile.close();

        if (Constants.FileStatus.END == fileBurstData.getStatus()) {
            return new FileBurstInstruct(Constants.FileStatus.COMPLETE);
        }

        // 文件分片传输指令
        FileBurstInstruct fileBurstInstruct = new FileBurstInstruct();
        fileBurstInstruct.setStatus(Constants.FileStatus.CENTER);            
        fileBurstInstruct.setClientFileUrl(fileBurstData.getFileUrl());      //客户端文件URL
        fileBurstInstruct.setReadPosition(fileBurstData.getEndPos() + 1);    //读取位置

        return fileBurstInstruct;
     }
}

    实现了读取写入文件的工具类后，就可以开始客户端的文件分片功能了。

    /*
ChannelInboundHandlerAdapter是 **处理器 需要继承的一个父类适配器，然后具体的 实现逻辑是在channelRead()方法中实现。
*/
public class MyClientHandler extends ChannelInboundHandlerAdapter {
    @Override
public void channelRead(ChannelHandlerContext ctx, Object msg) throws Exception {
	//数据格式验证
	if (!(msg instanceof FileTransferProtocol)) return;

	FileTransferProtocol fileTransferProtocol = (FileTransferProtocol) msg;
	//0传输文件'请求'、1文件传输'指令'、2文件传输'数据'
	switch (fileTransferProtocol.getTransferType()) {
		case 1:
			FileBurstInstruct fileBurstInstruct = (FileBurstInstruct) fileTransferProtocol.getTransferObj();
			//Constants.FileStatus ｛0开始、1中间、2结尾、3完成｝
			if (Constants.FileStatus.COMPLETE == fileBurstInstruct.getStatus()) {
				ctx.flush();
				ctx.close();
				System.exit(-1);
				return;
			}
			FileBurstData fileBurstData = FileUtil.readFile(fileBurstInstruct.getClientFileUrl(), fileBurstInstruct.getReadPosition());
			ctx.writeAndFlush(MsgUtil.buildTransferData(fileBurstData));
			break;
		default:
			break;
	}
}

    服务端中 实现分片传输和断电重续的重要逻辑

    channelRead处理文件协议，并且包含了保存续传信息，用于断点续传。

    // 当然这里处理器也是需要 继承 ChannelInboundAdapter 该类的 从而重写channelRead()方法
public class MyServerHandler extends ChannelInboundHandlerAdapter {
    @Override
    public void channelRead(ChannelHandlerContext ctx, Object msg) throws Exception {
        // 数据格式验证
    	if(!(msg instanceof FileTransferProtocol)) return;
    	FileTransferProtocol fileTransferProtocol = (FileTransferProtocol) msg;
        // 0传输文件'请求'、1文件传输'指令'、2文件传输'数据'
        switch(fileTransferProtocol.getTransferType()){
            case 0:
                FileDescInfo fileDescInfo = (FileDescInfo)fileTransferProtocol.getTransferObj();
                // 断点续传消息，实际应用中需要将断点续传的消息保存到数据库
                // 通过 文件描述对象 的名称 从自定义的缓存中获取 旧的分片指令对象
                FileBurstInstruct fileBurstInstructOld = CacheUtil.burstDataMap.get(fileDescInfo.getFileName());
                if(null != fileBurstInstructOld) {
                    // 传输完成 删除断点信息
                    if(fileBurstInstructOld.getStatus() == Constants.FileStatus.COMPLETE) {
                        CacheUtil.burstDataMap.remove(fileDescInfo.getFileName());
                    }
                    System.out.println(new SimpleDateFormat("yyyy-MM-dd HH:mm:ss").format(new Date()) + " 服务端，接收客户端传输文件请求[断点续传]。" + JSON.toJSONString(fileBurstInstructOld));
                    // 将旧的分片指令对象 利用MsgUtil构建成文件传输协议 给通道传输过去
                    ctx.writeAndFlush(MsgUtil.buildTransferInstruct(fileBurstInstructOld));
                    return;
                }
                // 给客户端发送消息
                FileTransferProtocol sendFileTransferProtocol = MsgUtil.buildTransferInstruct(Constants.FileStatus.BEGIN, fileDescInfo.getFileUrl(), 0);
                ctx.writeAndFlush(sendFileTransferProtocol);
                break;
            case 2:
                FileBurstData fileBurstData = (FileBurstData) fileTransferProtocol.getTransferObj();
                FileBurstInstruct fileBurstInstruct = FileUtil.writeFile("E://", fileBurstData);
                // 保存断点续传消息
                CacheUtil.burstDataMap.put(fileBurstData.getFileName(),fileBurstInstruct);
                ctx.writeAndFlush(MsgUtil.buildTransferInstruct(fileBurstInstruct));
                 //传输完成删除断点信息
                if (fileBurstInstruct.getStatus() == Constants.FileStatus.COMPLETE) {
                    CacheUtil.burstDataMap.remove(fileBurstData.getFileName());
                }
                break;
            default:
                break;
        }
        
    }
}

    # 总结

    首先我们得想为什么想要实现该功能，其目的是：加快一些大的文件的传输性能和稳定性，即使文件在中途因某种原因网络断开连接而无法上传，再次连接网络的时候还是可以在上次断开连接时那个上传进度 开始重新上传，而不需要重新从0开始上传。

    要想实现该功能，你的思路是如何的呢？（这里只是讲了我认为是核心的内容）

    1. 首先得定义好传输的协议：文件传输协议（FileTransferProtocol），该协议中包含传输类型和传输的数据对象；之后就开始定义数据传输对象：文件描述信息（FileDescInfo）、文件分片指令（FileBurstInstruct，该类作用是将 客户端想要传输的文件 转换为 传输协议对象）、文件分片数据（FileBurstData，该类作用是 存储文件分片传输的数据）；

    2. 定义完上述的协议后，开始构建 文件转为传输协议的媒介工具类MsgUtil类和文件读取写出工具类FileUtil类；其中文件读写工具类中结合Java中 RadomAccessFile 操作文件类实现了 文件分片读取写入的功能；

    3. 实现完了上述的铺垫后，开始客户端和服务端的文件操作：客户端的ChannelRead()中，主要是利用FileUtil.readFile() 方法将 读取到的文件地址 和 文件读取的初始位置 传输到Channel管道；

    4. 服务端根据传输协议中的传输类型来判断进行哪些操作fileTransferProtocol.getTransferType()，如果是0，是发送文件 ' 请求 '，首先在缓存（Map）中根据key文件的名称来查询是否有这个 文件名称的分片指令（value），如果有且判断该文件分片指令状态不是COMPLETE，则需要用MsgUtil将该指令转换为协议对象在channel管道传送过去；如果是2，是文件传输数据，则先保存断点续传的信息，即保存分片指令的名称（key）和分片指令（value），待到完成删除的断点信息后，就可以在缓存中移除这个指令的名称的内容了。
