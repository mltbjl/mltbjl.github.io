    01背包问题 以及 完全背包问题

    考小拉 9/7/2023 数据结构与算法hot 100 & 牛客 系列

    分割等和子集（背包是否能装满？）

    dp数组的含义：子集分为两个平均的集合；从[0,i]中任意取数据，背包所能装的数据的价值为 dp[i][j] ，如果能装满，则返回true，否则返回false。

    // 二维数组的递归公式
// 取：
dp[i][j] = dp[i - 1][j - weight[i]] + value[i];  
// 不取
dp[i][j] = dp[i - 1][j];

    目标和 （背包装满的方式有多少种？）

    也是将数组分成两类集合，一类是装正数的集合 left组合，一类是装负数的集合 right组合，left - right = target；

    而 left + right = sum；我们只需要求出装满其中一个集合方式个数，就可以知道装满另一个集合的方式个数；

    比如[1,1,1,1,1] target = 3；(target + sum) / 2 = 4；即求出当背包容量为4的时候，有多少种方式可以将背包填满，就可以求出target为3时有多少种方式了。

    这里更好的一种理解方式是一维数组，简单又好理解

       dp[0] = 1;
        // 先遍历物品，后遍历背包，而且背包要倒叙遍历（为什么？因为第一个，我们说了背包要倒叙遍历，如果先遍历背包后遍历物品的话，一个背包只能装一个物品而已；
        // 那为什么背包要倒叙遍历呢？因为是防止value[0]被放入两次）
        for (int i = 0; i < nums.length; i++) {

                // 求背包装满有几种方式 一般都是这个公式
                dp[j] += dp[j - nums[i]];

    0 和 1 （背包最多能装满之后的子集个数有几个？）

    这是 01背包问题，但是背包的维度有两个；一个是 m 一个是 n

    但是可以模仿地适用 01 背包的算法框架

     // 其实这也是类似于一维数组，代换过来的而已
        // 先遍历物品
        for (String str : strs) {
            // 统计 0 和 1的个数
            int zeroNum = 0, oneNum = 0;
            for (char ch : str.toCharArray()) {
                if (ch == '0') {
                    zeroNum++;
                } else {
                    oneNum++;
                }
            }

            // 后遍历背包，倒序遍历
            for (int i = m; i >= zeroNum; i--) {
                for (int j = n; j >= oneNum; j--) {
                    // 我们知道一维数组的 dp 值为 dp[j] = Math.max(dp[j],dp[j - weight[i]] + value[i]); 字符串的zeroNum和oneNum相当于物品的重量，而字符串本身的个数相当于物品的价值
                    dp[i][j] = Math.max(dp[i][j], dp[i - zeroNum][j - oneNum] + 1);
                }
            }
        }

    综上： 01 背包问题的相关力扣题目就是如此：

    1. 第一个我们需要将集合转换为 01 背包问题，01背包问题的特点：物品是具有价值和重量的；背包是具有容量的，一般这个容量是如何得到的？（01背包问题都是算 容器的大小的，比如说，分割等和子集求出一半大小的集合即可 集合大小就是一个背包容量；最后一个石头的重量II 分成两堆石头，那背包能装最多的石头嘛； 等等都是）

    2. 题目类型：

      • 分割等和子集：求的是 是否能装满背包的容量的问题

      • 最后一个石头的重量II：求的是 背包所能装的最大价值是多少：就可以用到我们的纯01背包的公式：

        // 不取当前：dp[j]  取当前的：dp[j - nums[i]] + nums[i]
dp[j] = Math.max(dp[j],dp[j - nums[i]] + nums[i]);
        1
        2

      • 目标和：求的是 装满背包的方式有多少种？

        // 这里也有一个公式 
dp[j] += dp[j - nums[i]];
        1
        2

      • 0 和 1：求的是 装满背包中所需要最多的物品是多少？ 虽然是一个背包，但是这个背包有两个维度。其实可以按照一个维度来写也是可以的；但是我们需要知道，求解这个问题的公式一般是，这里为什么是 + 1，和之前的 + value[i] 有啥关系嘛？其实就是一个求个数，一个求最大价值的区别。

        dp[j] = Math.max(dp[j],dp[j - nums[i]] + 1);
        1

    完全背包问题：跟 01 背包相比，一维数组的遍历来说，纯完全背包问题 背包和物品的遍历次序是可以调换的，而且都是从0开始进行遍历；大但是完全背包问题的 排列组合问题就不能这么说了。十分强调背包和物品的遍历次序问腿。

    // 排列问题（强调顺序，先遍历背包，后遍历物品）
for (int i = 1;i <= bagSize;i++) {
    for (int j = 0;j < nums.length();j++) {
        ....
    }
}
// 组合问题（不强调顺序，先遍历物品，后遍历背包）
for (int i = 0;i < nums.length();i++) {
    for (int j = 1;j < bagSize;j++) {

    单词拆分：物品是否能填满背包

    // dp数组的含义：长度为i的字符串，如果dp[i] = true，那就是集合中含有一个或者多个子串
boolean[] dp = new boolean[s.length() + 1]; 
// 因为本题 每个字符串的顺序是需要注意的，那么就是求排列数不是组合数，就需要先遍历背包后遍历物品
        for (int i = 1; i <= s.length(); i++) {
            for (int j = 0; j < i; j++) {
                String str = s.substring(j, i);
                // 这里的条件为什么是这样？当 set集合包含着这个子串，且该子串前的字符串都是true的情况下，那么截至到 i 这个索引的位置的字符串都是 符合题意的，就是 true
                if (set.contains(str) && dp[j]) {
                    dp[i] = true;
                    break;
                }
            }
        }
