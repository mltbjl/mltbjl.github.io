    深入理解volatile

    考小拉 3/9/2023 多线程编程Java基础

    直接开门见山，volatile是JVM中提供的轻量级的同步锁机制，volatile有哪些特性呢？

    • 保证预见性
    • 不保证原子性
    • 禁止指令重排

    以上三点必须要记住，但是记住总是不够的呀，会开车不代表你懂的车的底层原理（就不bb了~~）

    那么我们首先从保证预见性下手

    # 什么是预见性？

    这里我们需要知道JMM内存模型，那有人问什么是JMM内存模型（其实就是**Java内存模型**）？实话来说，JMM不是一个真是存在的东西，只是一个叫法或者说是一个规范，就像十二生肖中的龙一样，你能在现实生活中拉条龙出来吗？

    不废话，直接上图：

    如图：首先我们明白，所有变量都是存储在主内存中的；主内存中存储一个共享变量为77的值，然后三个线程都想对这个主内存的数据进行操作，那怎么操作呢？JVM会为每个线程都会开辟一个空间（名为工作内存），线程想对主内存的数值进行操作的话，需要对主内存的变量进行拷贝到工作内存中；然后进行修改，然后再将值传回给主内存进行修改。

    那问题又来了：比如上述的T1改了数值为87了，而且传回给主内存了，但是线程T2和T3却不知道改了这个数值呀，所以他还是用的数值还是之前的主内存的数值77，那么如何解决这个所谓的“线程通信” 的问题呢？这也就是所谓的预见性了。

    下面我们用代码来演示一遍这个volatile保证预见性的问题：

    class Data {
    int num = 0;
    public void addTo60() {
        this.num = 60;
    }
}
public class Test {
    public static void main(String[] args) throws InterruptedException {
        Data date = new Data();
        new Thread(() -> {
            try {
                Thread.sleep(3000);
            } catch (InterruptedException e) {
                throw new RuntimeException(e);
            }
            date.addTo60();
            System.out.println("现在num是" + date.num + "啦~");
        }).start();

        while (date.num == 0) {

        }
        System.out.println("现在num不是0啦~");
        System.out.println(date.num);
    }
}
/*
运行结果为：
现在num是60啦~（但是线程没有停止，陷入了死循环了）
*/

    # volatile如何不能保证原子性

    首先，解决这个问题之前，我们要了解什么是原子性：不可分割性，完整性，也就是在某个业务中，中间不可以被加塞或者分割，需要整体完整，要么同时成功，要么同时失败。

    那么下面我们用代码来验证volatile如何不能保证原子性的：

    生成20个线程，每个线程操作一个数200次，那么按常理来说，最后的结果肯定是4000次的，根据这个需求我们写代码：

    class MyData {
    volatile int num = 0;
    public void addNum() {
       num++;
    }
}

public class Test {
    public static void main(String[] args) throws InterruptedException {
        MyData myData = new MyData();
        for (int i = 0; i < 20; i++) {
            new Thread(() -> {
                for (int j = 0; j < 200; j++) {
                    myData.addNum();
                }
            }).start();
        }
        // 假如运行的线程还超过2 main线程让出cpu给其他线程继续执行
        while (Thread.activeCount() > 2) {
            Thread.yield();
        }
        System.out.println(myData.num);
    }
}

        atomicInteger.getAndIncrement();

}
public class Test {

        MyData myData = new MyData();

            new Thread(() -> {
                for (int j = 0; j < 200; j++) {
                    myData.addMyAtomic();
                }
            }).start();
        }
        while (Thread.activeCount() > 2) {
            Thread.yield();
        }
        System.out.println(myData.atomicInteger);
    }
}

    # 有序性

    在Java中，使用synchronized或者volatile保证多线程之间操作的有序性。实现原理的区别：

    • volatile关键字使用的是内存屏障达到禁止指令重排，保证有序性；
    • synchronized关键字使用的是一个JVM层面的monitor对象（monitorEnter和monitorExit）来进行操作的。被synchronized包住的代码块在多线程相当于串行化执行。

    # volatile如何禁止指令重排

    首先我们要清楚什么是指令重排：也就是虽然你写的程序是1 2 3这样的按顺序去写的，但是JVM读取的顺序也许不是这样的，这就是指令重排。

    举一个例子：

    public class ReSortSeqDemo {
    int a = 0;
    boolean flag = false;

    public void method01() {
        a = 1;
        flag = true;
    }
    public void method02() {
        if (flag) {
            a = a + 5;
            System.out.println(a);
        }
    }
}

    所以需要在每个操作属性上都需要加 volatile来保证在多线程情况下不发生指令重排。

    volatile int a = 0;
volatile boolean flag = false;

    典型的单例模式中有双重校验锁该类就用了volatile 来保证对象是单例的。

    public class Singleton {
    private static volatile Singleton singleton;
    private Singleton() {}
    public static Singleton getNewInstance() {
        if (singleton == null) {
            // 锁住类
            synchronized (Singleton.class) {
                if (singleton == null) {
                    /**
                     * 这里可以禁止指令重排
                     * 1. 分配了内存
                     * 2. 对对象进行初始化
                     * 3. 将对象的指针指向该内存空间
                     *
                     * 但是如果没有volatile，执行的顺序可能是132而不是123
                     */
                    singleton = new Singleton();
                }
            }
        }
        return singleton;
    }
}

    volatile关键字禁止指令重排有两层意思：

    • 当程序执行到volatile变量的读操作或者写操作时，在其前面的操作的更改肯定全部执行完成，其结果已经对后面的操作可见，在其后面的操作肯定没有执行；
    • 在进行指令优化的时候，不能将 在对volatile变量访问的语句放在其后面执行，也不能将volatile变量后面的语句放在其前面执行。

    比如：

    private static int a;//非volatile修饰变量
private static int b;//非volatile修饰变量
private static volatile int k;//volatile修饰变量

private void hello() {
    a = 1;  //语句1
    b = 2;  //语句2
    k = 3;  //语句3
    a = 4;  //语句4
    b = 5;  //语句5
    //以下省略...
}

    # 八种内存交互操作

    • lock（锁定），作用于主内存中的变量；
    • read（读取），作用于主内存中的变量；它把store操作从工作内存中得到的变量的值放入主内存的变量中。
    • load（加载），作用于工作内存中的变量；把read操作主存的变量放入到工作内存的变量副本中。
    • use（读取），作用于工作内存中的变量；
    • assign（赋值），作用于工作内存中的变量；
    • store（存储），作用于工作内存中的变量；它把一个从工作内存中一个变量的值传送到主内存中，以便后续的write使用。
    • write（写入），作用于主内存中的变量；它把store操作从工作内存中得到的变量的值放入主内存的变量中。
    • unlock（解锁），作用于主内存中的变量；

    # volatile禁止指令重排的原理是啥

    这里就涉及到了内存屏障，内存屏障分为几类：（了解一下）

    • LoadLoad屏障：

      Load1、LoadLoad，Load2。在Load2及后续读取操作要读取的数据被访问前，保证Load1要读取的数据被读取完毕。

    • StoreStore屏障：

      Store1、StoreStore、Store2。在Store2及后续写入操作执行前，保证Store1的写入操作对其他处理器可见。

    • LoadStore屏障：

      Load1， LoadStore，Store2。在Store2及后续写入操作被刷出前，保证Load1要读取的数据被读取完毕。

    • StoreLoad屏障：

      Store1， StoreLoad，Load2。在Load2及后续所有读取操作执行前，保证Store1的写入对所有处理器可见。

    在每个volatile读操作后插入LoadLoad屏障，在读操作后插入LoadStore屏障；

    在每个volatile写操作的前面插入一个StoreStore屏障，在每个volatile写操作后面插入一个StoreLoad屏障；
