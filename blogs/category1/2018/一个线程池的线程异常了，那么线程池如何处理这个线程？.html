    一个线程池的线程异常了，那么线程池如何处理这个线程？

    考小拉 10/7/2023 多线程编程线程池

    该篇文章参考 why技术 (opens new window)

    # 前言

    本文说的线程池都是 Executors 线程池；那么我们在了解这个问题之前，首先要知道，JDK 中默认提供了哪些线程池？

    • newFixedThreadPool(int nThreads)：线程池创建了固定大小的线程池，核心线程数量和最大线程数量都是同一个值；
    • newCachedThreadPool()：创建一个可缓存的线程池，核心线程数为0，最大线程数为 Integer.MAX_VALUE；适用于执行大量的短期任务；
    • newSingleThreadScheduleExecutor()：创建一个单线程的线程池，核心线程数为1，适用于需要保证任务顺序执行的场景
    • newScheduledThreadPool(int corePoolSize)：核心线程数自定义，最大线程数为 Integer.MAX_VALUE；可以执行定时任务和周期性任务；
    • newWorkStealingPool(int parallelism) ：创建一个工作窃取线程池，核心线程数为指定数量，适用于执行耗时较长的任务；

    # 进入正题

    先说预先的结论：

    • 抛出堆栈异常信息（对一半）
    • 不影响其他线程（全对）
    • 这个线程会回收到线程池（全错）

    我们先来看第一点：抛出堆栈信息为什么是对一半？

    • 测试用例

      public class Main {
    public static void main(String[] args) throws InterruptedException {
        ThreadPoolTaskExecutor executorService = buildThreadPoolTaskExecutor();
        executorService.execute(() -> sayHi("execute"));
        TimeUnit.SECONDS.sleep(2);
        System.out.println("========================");
        executorService.submit(() -> sayHi("submit"));
    }

    private static void sayHi(String name) {
        String printStr = "【thread-name:" + Thread.currentThread().getName() + ",执行方式:" + name + "】";
        System.out.println(printStr);
        throw new RuntimeException(printStr + ",我异常啦!哈哈哈!");
    }

    private static ThreadPoolTaskExecutor buildThreadPoolTaskExecutor() {
        ThreadPoolTaskExecutor executorService = new ThreadPoolTaskExecutor();
        executorService.setThreadNamePrefix("(刘东生测试)-");
        executorService.setCorePoolSize(5);
        executorService.setMaxPoolSize(10);
        executorService.setQueueCapacity(1000);
        executorService.setKeepAliveSeconds(30);
        executorService.setRejectedExecutionHandler(new ThreadPoolExecutor.CallerRunsPolicy());
        executorService.initialize();
        return executorService;
    }
}

    可以看到我们的 submit() 方法也有抛出堆栈异常消息了。

    所以说，线程异常的时候，不一定就是抛出异常消息，而是需要看你调用的是啥方法，如果是 execute() 方法，就直接可以抛出异常即可；但是如果是 submit() 方法就需要你得到该方法的返回值 Future<?> 然后再调用 get() 才能抛出异常信息。

    再来看第二点：不影响其他线程（全对）

    按照惯例，我们先写一个测试用例：

    public class Main {
    public static void main(String[] args) throws InterruptedException {
        ThreadPoolTaskExecutor executorService = buildThreadPoolTaskExecutor();
        executorService.execute(() -> sayHi("execute"));
        executorService.execute(() -> sayHi("execute"));
        executorService.execute(() -> sayHi("execute-exception"));
        executorService.execute(() -> sayHi("execute"));
        executorService.execute(() -> sayHi("execute"));
    }

    private static void sayHi(String name) {
        String printStr = "【thread-name:" + Thread.currentThread().getName() + ",执行方式:" + name + "】";
        if ("execute-exception".equals(name)) {
            throw new RuntimeException(printStr + ",我异常啦!哈哈哈!");
        } else {
            System.out.println(printStr);
        }

    }

    private static ThreadPoolTaskExecutor buildThreadPoolTaskExecutor() {
        ThreadPoolTaskExecutor executorService = new ThreadPoolTaskExecutor();
        executorService.setThreadNamePrefix("(刘东生测试)-");
        executorService.setCorePoolSize(5);
        executorService.setMaxPoolSize(10);
        executorService.setQueueCapacity(1000);
        executorService.setKeepAliveSeconds(30);
        executorService.setRejectedExecutionHandler(new ThreadPoolExecutor.CallerRunsPolicy());
        executorService.initialize();
        return executorService;
    }
}

    # 总结

    一个线程池的线程异常了，线程池会如何处理这个线程：

    1. 当执行方式是 execute 时,可以看到堆栈异常的输出；
    2. 当执行方式是 submit 时,堆栈异常没有输出。但是调用 Future.get() 方法时，可以捕获到异常；
    3. 不会影响线程池里面其他线程的正常执行；
    4. 线程池会把这个线程移除掉，并创建一个新的线程放到线程池中；
