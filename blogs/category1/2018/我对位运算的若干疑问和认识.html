    我对位运算的若干疑问和认识

    考小拉 8/10/2022 位运算

    1.首先位运算是什么？

    2.位运算能干些什么？

    3.为什么使用位运算而不用其他的运算？

    针对上面我的个人的几个问题，我冲浪和看书搜集一些资料并作出以下整理。

    # 首先位运算是什么？

    在我实际写代码中接触的位运算中，位运算符是一种计算符号。更加直接来说，是一种专门计算二进制的运算符，比如 1111 （15）若向右移一（两）位，即变成了 0111 （7） （0011 3）；若向左移动一（两）位，即变成了 11110 (30）（111100 60）。 由上述的计算其实我们可以先得出一个小的结论：倘若二进制数向右移动几位，就是将原来的数除以2的几次幂；倘若二进制数向左移动几位，就是将原来的数字乘2的几次幂（可能有人疑惑，1111 向右移动两位是3.5呀，怎么是3。因为int取值是向0取整的）。

    维基百科中的解释是：位操作是对数组或者二进制数的一元和二元的操作。

    即位运算就是基于整数的二进制表示进行的运算。

    # 位运算能干些什么？

    位运算就是来进行二进制运算的一种数学符号，更加底层来讲，是一种比加减乘除效率还要高的一种运算。位运算有这么几种操作符（& 与，| 或 ，^ 异或，~ 取反，>> 右移，<< 左移 ）

    # 为什么使用位运算而不用其他的运算？

    维基百科说到：在许多古老的微处理器上，位运算比加减运算略快，通常位运算比乘除法运算要快很多。在现代架构中，位运算的运算速度通常与加法运算相同（仍然快于乘法运算），但是通常功耗较小，因为资源使用减少。（对于如何做这个实验还望各位大牛指点）

    # 既然用到位运算，它操作符如何使用，应用场景是什么？

    # & 与运算符

    规则为：两个位数都为1时，才为1

    /**
 * & 运算规则为：两个位数都为1时，才为1
 * 0b 代表二进制
 * 101 --> 5
 * 100 --> 4
 * 两者进行 & 运算后，变为 100 --> 4
 */
int aa = 0b101;
int bb = 0b100;
System.out.println(aa & bb); // 4
//同理八进制也如此
//101 & 100 --> 100 64
int cc = 0101;
int dd = 0100;
System.out.println(cc & dd);//64
//16进制
//101 & 100 --> 100 256
int ee = 0x101;
int ff = 0x100;
System.out.println(ee & ff);//256

    用途：

    1.清零

    如果想将一个单元清零，即使其全部二进制位为0，只要与一个各位都为零的数值相与，结果为零。

    2.取一个数的指定位

    比如取数 X=1010 1110 的低4位，只需要另找一个数Y，令Y的低4位为1，其余位为0，即Y=0000 1111，然后将X与Y进行按位与运算（X&Y=0000 1110）即可得到X的指定位。

    int ii = 0b10101110;
int jj = 0b00001111;
String s = Integer.toBinaryString(ii & jj);
System.out.println(s); // 1110

    3.判断奇偶

    只要根据最未位是0还是1来决定，为0就是偶数，为1就是奇数。因此可以用if ((a & 1) == 0)代替if (a % 2 == 0)来判断a是不是偶数。

     public static void test(int i) { // 0011  3
        if ((i & 1) == 0) { //1:0001
            System.out.println(i + "是偶数");
        } else {
            System.out.println(i + "是奇数");
        }
    }

    # | 或运算符

    规则为：两个位数，有1则表示为1,否则为0

     /**
  * | 运算规则为：两个位数，有1则表示为1,否则为0
  * 0b 代表二进制
  * 101 --> 5
  * 100 --> 4
  * 两者进行 | 运算后，变为 101 --> 5
  */
int aa = 0b101;
int bb = 0b100;
System.out.println(aa | bb);
//同理八进制也如此
//101 | 100 --> 101 65
int cc = 0101;
int dd = 0100;
System.out.println(cc | dd);//65
//16进制
//101 | 100 --> 101 257
int ee = 0x101;
int ff = 0x100;
System.out.println(ee | ff);//257

    # ^ 异或运算符

    运算规则：两个位数，不同才为1，同则为0（我是这样去记的：相同的全没了（即为0），不同才留下（即为1）。）

    /**
 * ^ 运算规则为：两个位数，不同才为1，同则为0
 * 0b 代表二进制
 * 111 --> 7
 * 100 --> 4
 * 两者进行^运算后，变为011 --> 3
 */
int aa = 0b111;
int bb = 0b100;
System.out.println(aa ^ bb);//3
//同理八进制也如此
//111 ^ 100 --> 011 9
int cc = 0111;
int dd = 0100;
System.out.println(cc ^ dd);//9
//16进制
//111 ^ 100 --> 011 257
int ee = 0x111;
int ff = 0x100;
System.out.println(ee ^ ff);//17

    2.交换两个数

    public static void Swap(int a,int b){ //传入1 2
        System.out.println("交换前：");
        System.out.println(a+"\t"+b);
        if(a!=b){   //a = 1  b = 2
            a^=b;  // a:0001  b:0010
            b^=a;  // a:0011  b:0010  ----> 0001  b=1
            a^=b;  // a:0011  b:0001  ----> 0010  a=2
        }
        System.out.println("交换后：");
        System.out.println(a+"\t"+b);
}

/*输出结果为 
交换前：
1 2
交换后：
2 1
*/

    # << 左移运算符 >>右移运算符

    运算规则：倘若二进制数向右移动几位，就是将原来的数除以2的几次幂；倘若二进制数向左移动几位，就是将原来的数字乘2的几次幂

    //这里以二进制为例
int bb = 0b100; //4
System.out.println(bb >> 1);//2
System.out.println(bb << 1);//8
System.out.println(bb >> 2);//1
System.out.println(bb << 2);//16

    # ~ 取反运算符（区别于逻辑非）

    运算规则：二进制数按位取反 1 取反为 0，0取反为1，结果等于该值的补码减一。

    int gg = 0b10101011 //十进制171
/* 
该二进制取反后为0b01010100 等价于171的（补码-1） 
171的补码为 
先将10101011取反为01010100（反码），反码+1=补码
01010101则为171补码
*/
