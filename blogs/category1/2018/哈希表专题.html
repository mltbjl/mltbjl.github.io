    哈希表专题

    考小拉 9/1/2023 数据结构与算法hot 100 & 牛客 系列

    记住一个核心：想要判断一个字符是否出现在集合里，就用哈希表！

    # 有效的字母异位词

    力扣题目链接 (opens new window)

    为什么用数组作为哈希表呢？因为是给定的字符串，长度一定了，所以可以用数组代替哈希表。

    代码如下：

    public static boolean isAnagram(String s, String t) {
        if (s.length() != t.length()) return false;
        // 判断一个字符是否在集合中出现过，就用哈希表
        // 因为是固定长度，所以就可以用数组
        int[] arr = new int[26];
        for (int i = 0; i < s.length(); i++) {
            arr[s.charAt(i) - 'a']++;
        }
        for (int i = 0; i < t.length(); i++) {
            arr[t.charAt(i) - 'a']--;
        }
        for (int i = 0; i < arr.length; i++) {
            if (arr[i] != 0) return false;
        }
        return true;
}
/*
时间复杂度：O(n)
空间复杂度：O(1)
*/

    # 两个数组的交集

    力扣题目链接 (opens new window)

    这道题为什么用hashSet呢，因为假如用的是数组，num1 = {1,1,1000};不就是很浪费空间嘛？且hashSet有去重效果。

    public static int[] intersection(int[] nums1, int[] nums2) {
    Set<Integer> set = new HashSet<>();
    Set<Integer> resSet = new HashSet<>();
    for (int k : nums1) {
        // 往set添加元素，都是唯一的
        set.add(k);
    }
    for (int j : nums2) {
        if (set.contains(j)) {
        	resSet.add(j);
        }
    }
    // hashSet转数组的方法；List也是这样转数组的
    return resSet.stream().mapToInt(x -> x).toArray();
}
/*
时间复杂度：O(n)
*/

    # 数组中出现次数超过一半的数字

    牛客链接 (opens new window)

    题目思路：很直接的思路，遍历数组的时候，会统计每个数出现的次数，如果该数的次数为数组的一半则返回该数即可；

    public class Solution {
    public int MoreThanHalfNum_Solution (int[] numbers) {
        // write code here
        int[] arr = new int[10001];
        for (int i = 0; i < numbers.length; i++) {
            arr[numbers[i]]++;
            if (arr[numbers[i]] > numbers.length / 2) {
                return numbers[i];
            }
        }
        return -1;
    }
}

    # 数组中只出现一次的两个数字

    牛客链接 (opens new window)

    思路：遍历数组，map 来存储，如果 map 存在这个 key 就是证明我们原来就有这个数字的，那么就删除这个数字，后面直接处理我们的 map 即可

    public class Solution {
    public int[] FindNumsAppearOnce (int[] nums) {
        HashMap<Integer, Integer> map = new HashMap<>();
        for (int num : nums) {
            if (map.containsKey(num)) {
                map.remove(num);
            } else {
                map.put(num, 1);
            }
        }
        return map.keySet().stream().mapToInt(Integer::intValue).toArray();
    }
}

    # 缺失的正整数

    牛客链接 (opens new window)

    思路：先将我们数组中的正数遍历放进数组中，然后从 0 开始进行 while 循环遍历，每次都查看 map 中是否存在，不存在那就是答案了哈哈。

    public class Solution {
    public int minNumberDisappeared (int[] nums) {
        HashMap<Integer, Integer> map = new HashMap<>(16);

        for (int num : nums) {
            if (num > 0) {
                map.put(num, 1);
            }
        }
        int i = 0;
        while (true) {
            if (map.containsKey(i + 1)) {
                i++;
                continue;
            }
            return i + 1;
        }
    }
}

    # 两数之和

    力扣题目链接 (opens new window)

    该题目看起来很简单，但是做起来，每次做都会出问题（这里不考虑两层 for 循环解法）：思考的两个方向：①我们需要从哈希表中看是否有出现我们期待出现的数字，假如没有，也需要将该数字存在我们的哈希表中，并且还需要进行恢复我们的数字；②当找到我们期待出现的数字的时候，证明就是找到答案的时候了，收集即可；

     public static int[] twoSum(int[] nums, int target) {
        HashMap<Integer, Integer> hashMap = new HashMap();
        List<Integer> list = new ArrayList<>();
        for (int i = 0; i < nums.length; i++) {
            int result = target - nums[i];
            if (hashMap.containsKey(result)) {
                list.add(hashMap.get(result));
                list.add(i);
            } else {
                hashMap.put(nums[i], i);
            }
        }
        return list.stream().mapToInt(x -> x).toArray();
    }
/*
时间复杂度：O(n)
空间复杂度：O(n)
*/

    # 四数相加II

    力扣题目链接 (opens new window)

    这道题想要解决，应该首先将四数之和看作成是两数之和那样，一个存一个找。那么四个数组我们就想办法变成是 “两个数组“ 的操作即可。

    public int fourSumCount(int[] nums1, int[] nums2, int[] nums3, int[] nums4) {
        // key：数值 value：出现key的次数
        HashMap<Integer, Integer> map1 = new HashMap<>();
        int res = 0; // 记录结果
        for (int value : nums1) {
            for (int k : nums2) {
                int sum = value + k;
                if (map1.containsKey(sum)) {
                    map1.put(sum, map1.get(sum) + 1);
                } else {
                    map1.put(sum, 1);
                }
            }
        }

        for (int k : nums3) {
            for (int i : nums4) {
                int sum = k + i;
                System.out.println(sum);
                if (map1.containsKey(-sum)) {
                    // 这里不是res++，是因为等同于sum值个数不止一个，而是value值那个多个
                    res += map1.get(-sum);
                }
            }
        }
        return res;
    }
/*
时间复杂度：O(n*n)
空间复杂度：O()
*/

    # 字母异位词分组

    力扣题目链接 (opens new window)

    这道题目有几个难点：

    第一：如何判断字符串为字母异位词？

    第二：如何将字母异位词分在一个集合中？

    官方题解还挺有意思的，让我们排序后的字符串作为 key，然后用一个 ArrayList 作为 value；如果 key 相同的时候就可以获取到我们原本的 ArrayList 了，然后可以进行添加元素了。

    class Solution {
    public List<List<String>> groupAnagrams(String[] strs) {
        // 如何判断字符串为字母异位词

        // 如何将字母异位词分在一个集合中？用hashMap中存着我们的集合，等需要用到的时候再进行加入，没想过这个方法啊
        HashMap<String, List<String>> map = new HashMap<>(16);

        for (String str : strs) {
            char[] array = str.toCharArray();
            // 排序
            Arrays.sort(array);
            String s = new String(array);
            // 如果是存在这个key，就得到我们的value 即我们的集合来进行添加元素
            List<String> list = map.getOrDefault(s, new LinkedList<>());
            list.add(str);
            map.put(s, list);
        }

        return new LinkedList<>(map.values());
    }
}

    # 最长连续序列

    力扣题目链接 (opens new window)

    做这个题目的时候难点在于如何利用 O(n) 的时间复杂度来统计最长的连续序列嘛，这里我提供一个思路：见代码注释

    class Solution {
 	public int longestConsecutive(int[] nums) {
        if (nums.length == 0) {
            return 0;
        }
        Set<Integer> set = new HashSet<>();
        for (int num : nums) {
            set.add(num);
        }

        int res = 0;
        for (int num : nums) {
            /*
             这里为什么要判断集合中是否包含这个值呢？因为比如说 100 4 200 1 2 3 这个测试用例来说，如果你遇到了4，想要找3，然后以此向下寻找，直到找到了1，好，
             没问题，然后你遍历到了3的时候，就要找2，依次向下找，这样的话对是对的，但是复杂度高了呀，最坏的情况是可以达到 O(n ^ 2)的，所以我们干脆，比如说，遇到4时，
             如果集合中有3的话就跳过，等遇到了1时再也没有下一个值了，我们就进行开始升序计数即可啦。
             */
            if (!set.contains(num - 1)) {
                int cur = 1;
                int curNum = num;
                while (set.contains(curNum + 1)) {
                    curNum++;
                    cur += 1;
                }
                res = Math.max(res, cur);
            }
        }
        return res;
    }
}

    # 三数之和

    力扣题目链接 (opens new window)

    看到题目就得条件反射知道这题目用双指针来做；这道题的重点是去重；具体的流程可以看

    总共三个指针，其中一个指针是确定的值，就是nums[i] ，而其他遍历的两个指针就是所谓的 “双指针”。

    去重分为三个阶段：最好是记住这个示例 (-1, -1, -1, 2, 2)

    • 第一个是 nums[i]的去重，if( i > 1 && nums[i] == nums[i - 1]) continues; 这里为什么选择和nums[i - 1]比较而不是nums[i + 1] ；因为你的 left 指针后面指向的值也许会符合题目的条件（最后结合题目推演）；
    • 第二个去重就是nums[left] 和 nums[right]的去重了，做法也是和nums[i]一样。

    代码展示：（下面的j就是left）

    public List<List<Integer>> threeSum(int[] nums) {
        // 排序
        Arrays.sort(nums);

        List<List<Integer>> res = new ArrayList<>();
        int i;
        int right = nums.length - 1;

        for (int j = 0; j < nums.length; j++) {
            i = j + 1;
            // 去重
            if (j > 1 && nums[j] == nums[j - 1]) continue;
            int num = nums[i] + nums[j] + nums[right];
            while (j < right) {
                if (num < 0) {
                    j++;
                } else if (num > 0) {
                    right--;
                } else {
                    // 上面代码可以简洁代替
					res.add(Arrays.asList(nums[i],nums[j],nums[right]));
                    // 去重（取前面的 不要后面的）
                    while (j < right && nums[j] == nums[j + 1]) j++;
                    while (j < right && nums[right] == nums[right - 1]) right--;
                    j++;
                    right--;
                }
            }
        }
        return res;
    }

    # 四数之和（了解）

    力扣题目链接 (opens new window)

    在三数之和的基础上多了一个指针，如图

    那么就是比三数之和多了一层循环（就是多了第二指针的那层循环），做法也是和三数之和一样的了。

    代码如下：（加了剪枝操作）

     public List<List<Integer>> fourSum(int[] nums, int target) {
        List<List<Integer>> res = new ArrayList<>();
        Arrays.sort(nums); // 排序
        for (int i = 0; i < nums.length; i++) {
            // 剪枝
            if (nums[i] >= 0 && nums[i] > target) {
                break;
            }
            // 对num[i]去重
            if (i >= 1 && nums[i] == nums[i - 1]) {
                continue;
            }
            for (int j = i + 1; j < nums.length; j++) {
                if (nums[j] + nums[i] > target && nums[j] + nums[i] >= 0) {
                    break;
                }
                // 对num[j]去重
                if (j > i + 1 && nums[j] == nums[j - 1]) {
                    continue;
                }
                int left = j + 1;
                int right = nums.length - 1;
                while (left < right) {
                    int sum = nums[i] + nums[j] + nums[left] + nums[right];
                    if (sum > target) {
                        right--;
                    } else if (sum < target) {
                        left++;
                    } else {
                        res.add(Arrays.asList(nums[i],nums[j],nums[left],nums[right]));
                        // 对num[left] 和 num[right] 去重
                        while (left < right && nums[left] == nums[left + 1]) left++;
                        while (left < right && nums[right] == nums[right - 1]) right--;
                        right--;
                        left++;
                    }
                }
            }
        }
        return res;
    }
