    最大深度和最小深度其实是可以合起来做的，那样的话就可以避免最大的套路为什么用在最小的那里不行；就是说当我们遍历到该节点的时候，需要带入这个节点去考虑情况：其实是一个套路；

    // 最小
class Solution {
    public int minDepth(TreeNode root) {
        if (root == null) {
            return 0;
        }
        int left = minDepth(root.left);
        int right = minDepth(root.right);

        if (root.left != null && root.right == null) {
            return 1 + left;
        }
        if (root.right != null && root.left == null) {
            return 1 + right;
        }

        return Math.min(left, right) + 1;
    }
}
// 最大
class Solution {
    public int minDepth(TreeNode root) {
        if (root == null) {
            return 0;
        }
        int left = minDepth(root.left);
        int right = minDepth(root.right);

        if (root.left != null && root.right == null) {
            return 1 + left;
        }
        if (root.right != null && root.left == null) {
            return 1 + right;
        }

        return Math.max(left, right) + 1;
    }
}

    这三道题完全算是一个类型的题目；其中二叉搜索树嘛，那你按照中序遍历即可，只要数值不是递增的，那么就不是这个二叉搜索树；平衡二叉树主要是如果高度差大于1了，需要提前 return -1 了，不需要进行下面的递归了；完全二叉树就好办了，用层序遍历手段就可以了，因为完全二叉树所有的节点即使不是满二叉树，节点都是聚集在左端的。

    跟构造二叉树相关的

    这道题目的关键是，需要存储中序遍历的值，为什么呢？因为你需要靠这个来切割后序或者前序数组的值呀。知道了这个就很好做了题目；剩下的难度就是如何取这个边界值了。

    • [ ] 路径总和I

    前三道的题目思路都是一样的，都是利用回溯的套路；但是最后一道需要结合前缀和，其实就是需要用个map存储每个前缀和，因为求的是次数嘛，所以我们只需要得到以 curVal - target 为 value 的值即可，叠加。

    两个题目相同的思路，都是根据节点能贡献的最大值来进行计算；其中一个是值来计算。
