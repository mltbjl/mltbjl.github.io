    # 成员变量与局部变量的区别？

    这里我想了解的是 变量在内存中的存储方式如何？

    首先我们要知道 成员变量可以被 public、static、private等修饰符修饰，而局部变量不可以被访问修饰符比如说 public 和 static 修饰符修饰，但是都是可以被这个 final 修饰的；所以说，如果被 static 修饰的成员变量是属于类的，如果没有用 static 修饰，就是属于实例的，而我们又知道，这个类的信息是存储在我们的 方法区中的，我们的实例对象信息是存储在我们的 堆内存中的；而这个局部变量存储在我们的 栈内存中；

    # final 关键字的作用

    • 修饰在类上，类不可以被继承（String）
    • 修饰在方法上，方法不可以被重写
    • 修饰在变量上，变量的引用不可以变化；且必须要显式指定初始值。这里的不可以变化指的是变量的引用不可变，不是引用指向的内容不可以变化。

    # == 和 equals 的区别

    ==：作用是判断两个对象是否相等，即判断对象是否是同一个对象（如果是 基本数据类型，比较的是值；如果是 引用类型，比较的是 内存地址）

    equals：第一种情况：没有重写 equals() 方法，就是等同于 == ；如果是重写了，就是比较对象中的值是否相同；

    那问题来了，你重写过 hashcode 和 equals 么，为什么重写 equals 时必须重写 hashCode ⽅法？

    我们首先要了解什么是 hashCode，Java 中的 hashCode 是一个 Object 类的本地方法，通常都是将对象的内存地址转换为整数后进行返回。

    如果两个对象相等，则 hashcode 一定相等，两个对象相等，对两个对象进行调用 equals 方法都返回 true，反之，两个对象有相同的 hashcode 值，不一定回相等，hash 碰撞嘛。因此，equals ⽅法被覆盖过，则 hashCode ⽅法也必须被覆盖。

    hashcode() 方法默认行为是对堆上的对象产生独特值，如果没有重写 hashCode() ，则该 class 的两个对象无论如何都不会相等。

    # 深拷贝 & 浅拷贝

    首先两者都是创建对象的一种方式；

    • 浅拷贝：仅拷贝对象成员变量的值
    • 深拷贝：不仅对象成员变量的值会改变，同时也将引用变量指向堆中的实例也会拷贝

    图片展示 (opens new window)

    也可以记住这个图，深拷贝是安全的，浅拷贝的话如果你修改了引用类型，会影响原对象；

    浅拷贝可以使用我们的 Object 类中的 clone() 方法；

    深拷贝就需要重写我们的克隆方法，引用类型变量单独克隆，可能会涉及到多重递归；当然也可以进行序列化和反序列化；

    # String是不是基本数据类型？

    String 是一个比较特殊的引用类型；

    # String、StringBuilder、StringBuffer区别

    String：String 的值创建后不可以被修改，任何对 String 的修改都是会产生一个新的对象值，String 有很多 API，比如说 切割方法、分割方法、长度方法等；

    StringBuffer：跟 String 相似，但是值可以被修改，底层是使用这个 synchronized 来保证线程安全的；

    StringBuilder：StringBuffer 的非线程安全版，性能上更好一些；

    # intern 方法有什么作用

    官方解释：如果字符串内容（这里十分要注意！！！是字符串内容）存在于常量池（即 equals() 方法也为true，也是内容一样），直接返回字符串常量池中的字符串即可；否则就需要将此 String 对象添加到池中，并返回 String 对象的引用。

    # 反射

    我们创建对象都是通过 new 来进行获取的，编译时候就确定了这个类型信息；

    反射就是动态地获取一个类的信息、创建类的实例对象、调用类方法这时候就需要用到反射了；

    反射最核心的四个类：

    图片展示 (opens new window)

    • 应用场景？

    比如说我们的 Spring 的时候，一个 @Component 注解就可以声明一个 Bean 对象，通过一个 @Value 就可以获取配置文件的值，就是因为反射操作，然后到类/属性/方法/方法的参数上的注解，注解这里就有两个作用，一是标记，我们对注解标记的类/属性/方法进行对应的处理；二是注解本身有一些信息，可以参与到处理的逻辑中。

    • 反射原理

    我们知道 Java 程序的执行分为编译和运行两步，编译之后会生成字节码（.class）文件，JVM 进行类加载的时候，会加载字节码文件，然后将相关的所有信息加载到我们的方法区中，反射就是去方法区中获取到这样的信息然后进行操作的。
