    子串 & 子数组 专题

    考小拉 9/7/2023 数据结构与算法hot 100 & 牛客 系列

    最近这两天刷题目体会到一类题型：当题目求的是子串或者子数组的时候可以考虑这个前缀和以及滑动窗口来解决；如果是求区间和的时候可以考虑这个前缀和的思想来解决；当然也需要考虑一下动归。

    个人觉得前缀和的思想好想，但是做对应的题目还是不好做的；

    # 滑动窗口

    滑动窗口需要注意四个点：

    ① 扩大窗口时，如何进行更新操作？

    ② 什么条件下，窗口应该暂停更新，开始移动 left 缩小窗口？

    ③ 缩小窗口时，如何进行更新操作？

    ④ 我们要的结果应该在扩大窗口时还是缩小窗口时进行更新？

    • [ ] 长度最小的子数组
    • [ ] 无重复字符的最小子串
    • [ ] 找到字符串中所有字母的异位词
    • [ ] 最小覆盖子串
    • [ ] 滑动窗口的最大值

    最基础的滑动窗口的题目，第一题：

    • 长度最小的子数组 (opens new window)
    class Solution {
    public int minSubArrayLen(int target, int[] nums) {
        int i = 0, j = 0, sum = 0, subLen = 0, count = Integer.MAX_VALUE;
        while (i < nums.length) {
            // 右移动窗口时更新数据
            sum += nums[j];
            // 什么时候暂停更新，开始进行缩小窗口范围
            while (sum >= target) {
                // 截取数组长度
                // 获取的结果在我们进行收缩窗口的时候进行更新
                subLen = j - i + 1;
                if (subLen < count) {
                    count = subLen;
                }
                // 左移动来收缩窗口
                sum -= nums[i++];
            }
            j++;
        }

        return count == Integer.MAX_VALUE ? 0 : count;
    }
}

    • [无重复字符的最小子串](3. 无重复字符的最长子串 - 力扣（LeetCode） (opens new window))

    第一个找最长无重复子串嘛，肯定需要一个 map 进行记录元素的，这一步即扩大窗口时进行的数据更新操作嘛，当遍历元素的时候如果遇到了重复元素，此时就是需要暂停扩大窗口的操作，就需要进行这个缩小窗口的操作了；一般来说，缩小窗口的操作都是跟扩大窗口的操作是相反的模仿写就好了；然后到了是在扩大窗口的时候进行更新结果操作还是缩小窗口的时候进行更新操作的时候了，直接看代码即可；

    class Solution {
    public int lengthOfLongestSubstring(String s) {
        if (s.isEmpty()) {
            return 0;
        }
        if (s.length() < 2) {
            return 1;
        }
 HashMap<Character, Integer> map = new HashMap<>(16);
        int count = Integer.MIN_VALUE;

        int i = 0, j = 0;
        while (j < s.length()) {
            // c是将要移入滑动窗口的值
            char c = s.charAt(j);
            map.put(c, map.getOrDefault(c, 0) + 1);
            // 右移动窗口
            j++;
            
            // 判断左侧窗口是否需要移动
            while (map.get(c) > 1) {
                // 获取到左侧窗口的值
                char c1 = s.charAt(i);
                // 减去其即可
                map.put(c1, map.get(c1) - 1);
                i++;
            }
            // 此时 i 和 j 两者都 +1，所以相减的结果就是答案了
            if (j - i > count) {
                count = j - i;
            }
        }

        return count;
}
}

    • 找到字符串中所有字母的异位词 (opens new window)

    我们知道这个模板框架后，难点就在于如何进行收集元素以及这个扩大窗口以及缩小窗口后如何更新数据的操作了。

    class Solution {
    public List<Integer> findAnagrams(String s, String p) {
        // 记录s字符串中满足need哈希表记录的字符个数
        int count = 0;
        HashMap<Character, Integer> need = new HashMap<>(16);
        HashMap<Character, Integer> windows = new HashMap<>(16);
        for (int i = 0; i < p.length(); i++) {
            need.put(p.charAt(i), need.getOrDefault(p.charAt(i), 0) + 1);
        }
        List<Integer> res = new ArrayList<>();
        int i = 0, j = 0;
        while (j < s.length()) {
            // 即将进入窗口的值
            char c = s.charAt(j);
            j++;
            // 扩大窗口时，需要做哪些更新操作？
            // 如果need哈希表中包含了该字符，就需要进行累加count
            if (need.containsKey(c)) {
                windows.put(c, windows.getOrDefault(c, 0) + 1);
                // 如果窗口中含有该字符的数量等于我们need中含有的字符对应的数量，则进行count++
                if (windows.get(c).equals(need.get(c))) {
                    count++;
                }
            }

            // 判断是否需要收缩窗口（当我们窗口的长度大于这个目标子串的时候就需要进行缩小窗口）
            while (j - i >= p.length()) {
                // 结果是在缩小窗口的时候进行收集
                if (count == need.size()) {
                    res.add(i);
                }

                char d = s.charAt(i);
                i++;
                // 缩小窗口时，需要做哪些更新操作？
                // 如果need哈希表中包含了该字符，就需要进行减去
                if (need.containsKey(d)) {
                    // 如果窗口中含有该字符的数量等于我们need中含有的字符对应的数量，则进行count--
                    if (windows.get(d).equals(need.get(d))) {
                        count--;
                    }
                    windows.put(d, windows.get(d) - 1);
                }
            }

        }

        return res;
    }
}

    • 最长覆盖子串

    这道题难点在于 知道模板框架后如何进行这个收集元素 以及 扩大和缩小窗口后的更新操作是啥？我们这个题目是需要截取字符串的，那么我们就需要一个标记起始点，以及一个终点（这里有一个技巧：后面如果看到这种截取字符串长度的，尽量用一个变量去记录 start 起始点不要用这个 原始双指针去进行标记，终点的话也是可以进行 left + maxLen 标记 或者用一个变量 end 去标记终点）；

    public class Solution {
    public String minWindow (String S, String T) {
         HashMap<Character, Integer> need = new HashMap<>();
        HashMap<Character, Integer> window = new HashMap<>();

        for (int i = 0; i < T.length(); i++) {
            need.put(T.charAt(i), need.getOrDefault(T.charAt(i), 0) + 1);
        }

        int count = 0, left = 0, right = 0, maxLen = Integer.MAX_VALUE,start = 0;
        while (right < S.length()) {
            char c = S.charAt(right);
            right++;
            // 右移窗口后的更新操作
            if (need.containsKey(c)) {
                window.put(c, window.getOrDefault(c, 0) + 1);
                // 如果两个含有的数量是一样的
                if (window.get(c).equals(need.get(c))) {
                    count += need.get(c);
                }
            }

            while (count == T.length()) {
                // 收集结果的时候
                if (right - left < maxLen) {
                    maxLen = right - left;
                    // 标记起点
                    start = left;
                }
                char d = S.charAt(left);
                left++;

                if (need.containsKey(d)) {
                    if (window.get(d).equals(need.get(d))) {
                        count -= window.get(d);
                    }
                    window.put(d, window.get(d) - 1);
                }
            }
        }
        if (maxLen == Integer.MAX_VALUE) {
            return "";
        }
        return S.substring(start, start + maxLen);
    }
}

    • 滑动窗口的最大值 (opens new window)

    这道题目考察的是单调队列；可以参考一下单调栈；单调栈的特点就是 寻求当前该数左边或者右边的第一个比当前数大或者小的数（比如说：1 3 4 2；3左边第一个比3小的数是1，3右边第一个比3小的数是2）；单调队列可以这样理解：每次队列中维护的都是目标区间内可能是最大的数即可，有点难理解这里，而且我们队列中存储的是索引值而不是我们的真正的值（这点和我们的单调栈一样）

    class Solution {
    public int[] maxSlidingWindow(int[] nums, int k) {
        Deque<Integer> deque = new LinkedList<>();

        for (int i = 0; i < k; i++) {
            // 当队列不为空 && 队列头部元素小于 比较元素时，删除队列头部元素
            while (!deque.isEmpty() && nums[i] >= nums[deque.peekLast()]) {
                deque.pollLast();
            }

            deque.offerLast(i);
        }
        int[] res = new int[nums.length - k + 1];
        res[0] = nums[deque.peekFirst()];
        for (int i = k; i < nums.length; i++) {
            // 始终维护一个最大值在队列头部
            while (!deque.isEmpty() && nums[i] > nums[deque.peekLast()]) {
                deque.pollLast();
            }

            deque.offerLast(i);
            // 剔除过时的索引
            while (i - k >= deque.peekFirst()) {
                deque.pollFirst();
            }
            res[i - k + 1] = nums[deque.peekFirst()];
        }
        return res;
    }
}

    # 前缀和

    前缀和的思想就是当我们要求某个区间的和的时候可以使用这个方法；就是使用一个 前缀和 + HashMap 的思想来进行解题；但是问题来了，这个哈希里面存的是啥就比较考究了。。。。

    • [ ] 和为 K 的子数组

    定义 HashMap ，然后存的 key 为 当前索引的前缀和，value 则是我们的前缀和出现的次数；每次遍历的时候 preSum - K 是否存在我们的 map 中，存在则代表出现这个子数组。

    class Solution {
    public int subarraySumII(int[] nums, int k) {
        HashMap<Integer, Integer> map = new HashMap<>();
        int preSum = 0, res = 0;
        map.put(0, 1);
        for (int i = 0; i < nums.length; i++) {
            // 前缀和
            preSum += nums[i];
            if (map.containsKey(preSum - k)) {
                res += map.get(preSum - k);
            }
            map.put(preSum, map.getOrDefault(preSum, 0) + 1);
        }

        return res;
    }
}
