    # Kotlin 中匿名函数章节

    # 匿名函数定义

    定义时不取名字的函数，称之为匿名函数，匿名函数通常整体传递给其他函数，或者从其他函数返回；

    fun main() {
    val total = "Mississipi".count()
    println(total)
    
    // 找出total中s字符的个数
    val totalS = "Mississipi".count( {letter -> letter == 's'} )
    println(totalS)
}

    # 匿名函数类型隐式返回

    匿名函数也有类型，匿名函数可以当作变量赋值给函数类型变量，就像其他变量一样；变量有类型，变量可以等于函数，函数也会有类型；函数的类型，由传入的参数和返回值类型进行决定

    和具名函数不一样，除了极少数的情况，匿名函数不需要 return 关键字来进行返回数据，匿名函数会隐式或者自动返回函数体最后一个行语句的结果

    fun main() {
    // 1. 变量等于函数，函数的返回类型就是 String，传入参数为无参
    val blessingFunction:() -> String = {
        val holiday = "New Year"
        // 不需要 return 返回值，而是隐式地进行返回
        "Happy $holiday"
    }
    println(blessingFunction)
}

      这个函数可以看作是一个配置函数，可以传入一个接收者，然后调用一系列函数来配置它以便使用，如果提供 lambda 给 apply 函数执行，他会返回配置好的接收者。

    • also 函数

      also 函数和 let 函数功能相似，和 let 一样，also 也是把接收者作为值参传给 lambda.但有一点不同: also 返回接收者对象，而 let 返回 lambda 结果。因为这个差异，also 尤其适合针对同一原始对象，利用副作用做事，既然 also 返回的是接收者对象，你就可以基于原始接收者对象执行额外的链式调用。

      fun main() {
    var fileContents:File<String> // 没有初始化
    File("xxxx")
    .also {
        println(it.name)
    }.also {
        fileContents = it.readLines() // 初始化
    }
    
    println(fileContents)
}

    多泛型参数：泛型函数或者泛型类也可以有多个泛型参数

    还是上面那个案例：

    class MagicBox3<T>(item: T) {
    var available = false
    private var subject: T = item

    fun fetch(): T? {
        return subject.takeIf { available }
    }

    // 这个函数的业务需求意思是：我们这个 fetch 函数的参数是一个函数，这个函数的参数是一个泛型类型，就是我们将这个
    // 泛型类型的参数传入后，返回的是经过 subjectModFunction() 函数处理后的 R 类型的结果回去
    fun <R> fetch(subjectModFunction: (T) -> R): R? {
        return subjectModFunction(subject).takeIf { available }
    }
}

    val man = box11.fetch {
        Man(it.name, it.age.plus(10))

    想要让其他也可以实现，那么就需要继承这个 Human 类，当然你可以接收多个实例，使用 vararg 关键字。

    class MagicBox<T : Human>(vararg item: T) {
    var available = false
    private var subject: Array<out T> = item

    fun fetch(index: Int): T? {
        return subject[index].takeIf { available }
    }

    // 这个函数的业务需求意思是：我们这个 fetch 函数的参数是一个函数，这个函数的参数是一个泛型类型，就是我们将这个
    // 泛型类型的参数传入后，返回的是经过 subjectModFunction() 函数处理后的 R 类型的结果回去
    fun <R> fetch(index: Int, subjectModFunction: (T) -> R): R? {
        return subjectModFunction(subject[index]).takeIf { available }
    }
}

open class Human(val name: String)
class Boy(val name: String, val age: Int)
class Dog(val weight: Int)

fun main() {
    val box1: MagicBox<Human> = MagicBox(
        Human("刘东生"),
        Human("刘婉雯")
    )

    box1.available = true
    box1.fetch(1)?.run {
        println("you find $name")
    }
}

    继续，我们来看看 在 private var subject: Array<out T> = item 使用了 out，什么意思呢？

    out(协变)，如果泛型类只将泛型类型作为函数的返回（输出），那么使用 out，可以称之为生产类/接口，因为它主要是用来生产（produce）指定的泛型对象。

    in （逆变），如果泛型类只将泛型类型作为函数的入参（输入），那么使用 in，可以称之为消费者类/接口，因为它主要是用来消费（consume）指定的泛型对象

    我们可以写 Demo 来看看：

    // out
interface Production<out T> {
    fun produce(): T
}

// in
interface Consumer<in T> {
    fun consume(item: T)
}

interface ProductionConsumer<T> {
    fun product(): T
    fun consume(item: T)
}

open class Food
open class FastFood : Food()

class Burger : FastFood()

// 生产者，食品商店
class FoodStore : Production<Food> {
    override fun produce(): Food {
        println("produce food.")
        return Food()
    }
}

// 生产者，快餐商店
class FastFoodStore : Production<FastFood> {
    override fun produce(): FastFood {
        println("produce fastFood.")
        return FastFood()
    }
}

// 生产者，汉堡商店
class BurgerStore : Production<Burger> {
    override fun produce(): Burger {
        println("produce burger.")
        return Burger()
    }
}

// 消费者
class EveryBody : Consumer<Food> {
    override fun consume(item: Food) {
        println("consumer food.")
    }
}

class ModernPeople : Consumer<FastFood> {
    override fun consume(item: FastFood) {
        println("consumer fastFood.")
    }
}

class American : Consumer<Burger> {
    override fun consume(item: Burger) {
        println("consumer burger.")
    }
}


fun main() {
    val production1: Production<Food> = FoodStore()
    // 这里为什么 也是不会报错呢？在 Java 中泛型为父类的 不能指向 泛型为子类的，只能平级
    // 比如说：List<CharSequence> list = new ArrayList<String>(); 但是 kotlin 就可以，就源于这个 in out 关键字
    // 总结：子类泛型对象可以赋值给父类泛型对象，用 out
    val production2: Production<Food> = FastFoodStore()
    val production3: Production<Food> = BurgerStore()


    // 原理同上；总结：父类泛型对象可以赋值给子类泛型对象，用 in
    val consumer1: Consumer<Burger> = EveryBody()
    val consumer2: Consumer<Burger> = ModernPeople()
    consumer2.consume(Burger())
    val consumer3: Consumer<Burger> = American()
}

    println(animals)

    # Kotlin 和 Java 互相调用

    • @JvmName 来指定编译类的名字

      在 Kotlin 文件中，使用该注解来改名字，方便 Java 调用

    • @JvmField 来指定编译类中的属性

    • @JvmOverload 设计一个可能会暴露给 Java 用户使用的 API 时，就需要用到这个注解；因为在 kotlin 中，假设函数有两个参数，而且还有默认值，我们 kotlin 可以直接使用一个参数就够了，另一个是默认形参；但是 Java 不行，需要重载，所以说，这个注解的作用就体现在这里了；

      class Spellbook {

        @JvmStatic
